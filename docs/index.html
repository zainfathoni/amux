Amp runs your agents anywhere. amux keeps the local side — tmux windows, threads, runners — restorable from plain TSV files, with lifecycle verbs safe enough to hand to the agents themselves.

Every verb declares its blast radius.

amux separates four side-effect domains and each command touches the fewest it can. Click a verb — the terminal plays it, and the domains it mutates light up.

All releases →
amux completion <shell>

Static completions for bash, zsh, and fish — commands, aliases, flags, runner subcommands.

amux update

Self-update user-local installs, with --dry-run. Refuses Homebrew/Nix-managed paths so package managers stay in charge.

amux workspaces

Instant local workspace inventory, no Amp calls. Add --include-runners for machine-level checks.

amux prune-archived

Forget restore rows whose threads are confirmed archived. Fails closed on any doubt.

spawn --title-prefix '#255'

Issue-numbered workers: tmux window, restore row, and Amp thread all named #255 worker.

spawn --message-file …

Multi-line prompts from a file or stdin, delivered via bracketed paste and verified against the new thread.

Your agent already speaks it.

The bundled /amux skill teaches Amp agents the safe vocabulary — which verb mutates which domain, and when to run doctor instead of guessing.

$ ln -sfn "$PWD/skills/amux" ~/.agents/skills/amux

Skill workflows go beyond the CLI: read the skill source.

“Pin it”amux pin-current <thread-url>
“Park it”amux park-current
“Shelve this”amux shelve-current
“Restore my workspace”amux launch
“/amux sprawl #12 #34”one worktree + spawned worker per independent issue
“/amux health”read-only responsiveness check across workers
“/amux finish”post-merge git/GitHub cleanup, then verified teardown
