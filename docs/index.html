Local tmux workspaces for Amp

Restore agent workspaces without the ritual.

Amp can now start agents anywhere you run a runner. amux complements that with a small CLI and bundled /amux skill for keeping local Amp/tmux workspaces restorable, inspectable, and safe for agents to operate.

See the workflow in one minute.

amux is not another remote agent system. It is a tiny local contract between your tmux windows, Amp threads, and a TSV file you can inspect before an agent touches anything.

Built for persistent local agent work.

Amp Agents Anywhere provides first-party remote thread creation and per-directory runners. amux manages the local tmux workspace around the Amp sessions you want to keep organized.

Complement Agents Anywhere
Use Amp-native remote creation for new agents, and use amux for the local tmux workspace lifecycle: restore known threads, store runner intent, inspect drift, park, spawn, and verified cleanup.
Restore workspaces
Launch a named workspace into a tmux session with one window per configured Amp thread. One-workspace-argument commands use the same name for the tmux session; pass an explicit session such as Amp for older shared-session layouts.
Pin the current window
Pin or unpin the tmux window you are already in without manually retyping its name or path. These commands only mutate restore config.
Park live sessions
park and park-current stop only the resolved local tmux/Amp window while preserving the restore row and leaving remote Amp thread history untouched.
Shelve deferred work
shelve-current pins the current tmux/Amp window if needed, archives the identified thread, and stops the current window. shelve archives one existing restore row or a whole workspace so deferred work leaves the Amp sidebar while rows stay available for explicit unshelve later.
Inspect drift
Use doctor [workspace] [session] to compare thread restore rows and runner rows with the intended live tmux session before trusting a restore. For example, amux doctor amux checks session amux, while amux doctor mac Amp checks a legacy shared Amp session.
Manage runners separately
runner pin, runner launch, and runner park manage local amp --no-tui runner windows from runners.tsv, without faking thread IDs or touching remote Amp thread state.
Preview first
launch --dry-run and spawn --dry-run validate inputs and report intended actions, including planned title-prefix renames, without mutating restore config, local tmux/Amp state, or remote Amp threads.
Spawn fresh threads
Create a new Amp thread in a real interactive tmux window, optionally pass --mode, use --message-file or --message-stdin for explicit multi-line prompts, or name both the tmux window and new Amp thread <prefix> <window> with --title-prefix, then add it to your restore config and inject stable AMUX_* identity for lifecycle commands.
Teardown workers safely
amux teardown verifies spawned identity against config and tmux before archiving the matching Amp thread, removing the row, and stopping the uniquely matched window. From a restored worker without AMUX_*, use amux teardown --thread <thread-id-or-url> [--session <session>]; from outside the worker when you know the row, use amux teardown <workspace> <window> [session]. Both forms cross-check the restore row and live tmux start command agree on the same thread before mutating anything.
Plain-text config
The restore contract is a simple TSV file that is easy to read, diff, back up, and sync.
Agent skill included
The repository includes a /amux skill that teaches Amp agents the safe vocabulary: pin for restore config, park for live-local shutdown, teardown for verified full cleanup, and doctor before guessing.

A tiny contract you can understand.

The default config lives at ~/.config/amux/workspaces.tsv. Each row maps a workspace, a tmux window, a working directory, and an Amp thread.

amp --no-tui runners are per working directory. Runner intent lives separately in ~/.config/amux/runners.tsv, so amux runner launch can restore local runner windows without mixing them into thread restore rows.

amux names four side-effect domains explicitly: restore-config rows, runner-config rows, live local tmux/Amp sessions, and remote Amp thread state. Pin/unpin commands only change the restore config (store/remove remain compatibility aliases); runner pin/runner unpin only change runner config; park and runner park only stop resolved live local windows while preserving config; shelve-current pins/preserves the current row before archiving the supplied current thread; row-based shelve archives/hides deferred Amp threads while preserving restore rows; teardown is the explicit full cleanup that archives the verified thread, removes its row, and stops its verified local window.

amux list [workspace] keeps the restore TSV fields first and appends status: active, shelved, missing, or unknown when Amp status cannot be read. Use amux list --active, amux list --shelved, or the amux shelved shortcut to focus the view.

Config format

workspace   window       workdir             thread
mac         my-project   ~/Code/my-project   T-example
mac         docs         ~/Code/docs         T-docs

# runners.tsv
workspace   window             workdir
mac         project-runner     ~/Code/my-project
1 PinSave the current tmux window in restore config with amux pin-current.
2 RestoreRun amux launch to recreate the workspace later.
3 Shelve or teardownUse amux shelve-current [workspace] <thread-id-or-url> from an unpinned current pane, or amux shelve --thread <thread-id-or-url>/amux shelve --workspace <workspace> for existing rows, to hide deferred work while keeping rows restorable by explicit amux unshelve ...; use no-arg amux teardown only from a spawned AMUX_* identity, amux teardown --thread <thread-id-or-url> [--session <session>] from a restored worker without identity, or amux teardown <workspace> <window> [session] for full cleanup.
4 VerifyRun amux list --active [workspace], amux shelved [workspace], or amux doctor [workspace] [session] when live tmux state and config may have drifted.
5 Agent skillInstall skills/amux as ~/.agents/skills/amux so agents can respond reliably to phrases like “pin it”, “park it”, “teardown this worker”, and “doctor amux”.

Install it, try it, keep what works.

amux is intentionally small: a Go binary, tmux, Amp, and a text file. Start with a dry run, then let it restore the local Amp workspace you already use. 

$ brew tap zainfathoni/amux https://github.com/zainfathoni/amux
$ brew install zainfathoni/amux/amux
$ brew upgrade amux
